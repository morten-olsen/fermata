## Design rationale

The track row follows the **list row pattern** from the design system: a horizontal layout with leading element, flex text, and trailing metadata. Every track in every context uses this same component.

## States

### Default
<Canvas of={TrackRowStories.Default} />

### Now Playing
<Canvas of={TrackRowStories.NowPlaying} />

The title turns accent and the track number is replaced by animated equalizer bars.

### Favourited
<Canvas of={TrackRowStories.Favourited} />

A small accent heart appears. The heart is only visible when the track is favourited — no outline heart for unfavourited tracks (clean by default).

### Downloaded / Queued
<Canvas of={TrackRowStories.Downloaded} />
<Canvas of={TrackRowStories.Queued} />

Tiny cloud icons indicate download status — `cloud-done` for downloaded, `cloud-download-outline` for queued.

### Full featured
<Canvas of={TrackRowStories.FullFeatured} />

All indicators at once: downloaded, favourited, with a more button.

## Layout anatomy

```
┌──┬──────────────────────────┬──┬──┬────┬──┐
│##│ Title                    │☁ │♥ │3:42│⋯ │  py-3 px-1
│  │ Artist                   │  │  │    │  │
└──┴──────────────────────────┴──┴──┴────┴──┘
 32   flex-1                   optional  trailing
```

Each trailing element only renders when its prop is provided, keeping the row minimal by default.
