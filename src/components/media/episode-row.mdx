## Design rationale

Podcast episodes have fundamentally different browsing needs than music tracks. You scan by **date** (what's new?), care about **progress** (where did I leave off?), and don't need favourites or contextual menus.

## How it differs from TrackRow

| | TrackRow | EpisodeRow |
|---|---|---|
| **Subtitle** | Artist name | Publish date |
| **Favourite** | Heart toggle | None |
| **More menu** | "..." button | None |
| **Progress** | Optional bar | Built-in bar |
| **Leading** | Track number | Episode number or mic icon |
