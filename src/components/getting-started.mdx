## Import patterns

Each category has a **barrel file** as its public API. Import from the barrel, not individual files:

```tsx
// Good — barrel import
import { PressableScale } from "@/src/components/primitives/primitives";
import { MediaRow, MediaCard } from "@/src/components/data-display/data-display";
import { useBreakpoint } from "@/src/components/layout/layout";

// Bad — direct file import
import { PressableScale } from "@/src/components/primitives/pressable-scale";
```

## Compound components

Complex components use the **compound pattern** (like Radix UI). The root component has sub-components attached as properties:

```tsx
// Composable — full control
<MediaRow onPress={play} isPlaying>
  <MediaRow.Leading>
    <MediaRow.PlayingIndicator />
  </MediaRow.Leading>
  <MediaRow.Content title="Song" subtitle="Artist" />
  <MediaRow.Duration seconds={213} />
</MediaRow>

// Preset — standard layout in one line
<MediaRow.Track title="Song" artistName="Artist" duration={213} onPress={play} />
```

Both APIs are equivalent. Use presets for standard layouts, composable API for custom ones.
